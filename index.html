El Ecosistema Tecnológico

Arquitectura moderna orientada a servicios, gestión ágil y entrega de alta calidad

React 18

Frontend modular SPA con React Router, hooks personalizados y consumo de API asíncrono.

Spring Boot 3

Backend robusto en Java con arquitectura REST, JPA/Hibernate, Maven y seguridad JWT.

JSON Web Tokens

Seguridad y control de acceso basado en roles (ADMIN, OWNER, VET, CLINIC_OWNER).

PostgreSQL / H2

Base de datos relacional con persistencia optimizada y base de datos H2 en memoria para tests.

Docker

Contenedores e imágenes para simplificar el despliegue del entorno y bases de datos locales.

SonarQube

Análisis de código estático continuo para asegurar calidad de software, bugs 0 y alta cobertura.

ZenHub & Scrum

Planificación de Sprints con estimación en Story Points, tableros Kanban y Burndowns continuos.

iTop (ITSM)

Gestión y monitorización de incidencias, peticiones de servicio y Acuerdos de Nivel de Servicio (SLA).

Arquitectura y Estructura de Datos

Diseño de software robusto, limpio y escalable con separación de responsabilidades

React SPA (Frontend)

Vistas modulares, Axios/Fetch, Context API para estado de sesión, almacenamiento local de tokens JWT.

Peticiones HTTPS (JSON) + Token JWT
JWT Security Layer

Interceptor de peticiones y filtros de seguridad en Spring Security. Valida y autoriza roles del token.

Endpoints Autorizados
Spring Boot REST API

Controladores REST independientes (AdoptionController, BookingController, VetController). Capa de Servicios JPA.

JPA Hibernate
Base de Datos SQL

Tablas normalizadas con relaciones complejas. Entidades unificadas e indexadas para consultas rápidas.

owners
  • PK id SERIAL
  • first_name VARCHAR(30)
  • last_name VARCHAR(30)
  • address VARCHAR(255)
  • city VARCHAR(80)
  • telephone VARCHAR(20)
  • FK user_id INTEGER
pets
  • PK id SERIAL
  • name VARCHAR(30)
  • birth_date DATE
  • FK type_id INTEGER
  • FK owner_id INTEGER
bookings (Hotel)
  • PK id SERIAL
  • start_date DATE
  • end_date DATE
  • FK pet_id INTEGER
  • FK room_id INTEGER
  • status VARCHAR(20)
adoptions
  • PK id SERIAL
  • description TEXT
  • post_date TIMESTAMP
  • FK pet_id INTEGER
  • FK owner_id INTEGER
  • status VARCHAR(20)
